**BitBattles** — экспериментальный автобаттлер, в котором интеллект юнитов и тактика противника создаются с помощью гибридного подхода: обучение с подкреплением (Deep Q-Network) управляет поведением юнитов игрока, а генетический алгоритм эволюционирует расстановку врагов.

Курсовая работа студентов НИУ ВШЭ (Нижний Новгород): Айдара Секамова и Алана Келехсаева. Год защиты: 2026

## Ключевые особенности

## Как это работает

### RL-агент (управление юнитами игрока)

- Алгоритм: DQN с целевой сетью и Replay Buffer (50 000 записей)
- Нейросеть: 27 → 128 → 128 → 64 → 5 (выход: 5 действий)
- Входные данные: координаты, HP, уровень, тип, ближайший враг, 8 соседей, 9 клеток в радиусе атаки, укрытие за камнем
- Система наград: убийство врага (+5), подбор хилки (+3), укрытие за камнем (+0.5), смерть (-10)
- Результат: Win Rate около 52%

### GA-агент (расстановка врагов)

- Алгоритм: генетический алгоритм + нейросеть (нейроэволюция)
- Нейросеть: 102 → 64 → 32 → 225 (9 типов юнитов × 25 клеток, Sigmoid)
- Эволюция: популяция 15 особей, турнирная селекция (размер 4), равномерный кроссовер, гауссовская мутация, элитизм (топ-10%)
- Результат: Max Fitness 143.0, Win Rate до 86.7%

### Сетевое взаимодействие

- RL-сервер: порт 65432, JSON
- GA-сервер: порт 5005, строковый формат (бюджет|фитнес|состояние_поля)

## Установка и запуск

### Требования

- Windows 10/11 или macOS
- Unity 6000.0.25f1 (LTS)
- Python 3.10+ с пакетами: torch, numpy, matplotlib, notebook
- Разрешение экрана: 1920x1080 (рекомендуется)

### Шаги

1. Клонировать репозиторий

git clone https://github.com/Varied-Kratos/BitBattles.git
cd BitBattles

2. Запустить RL-сервер (в отдельном терминале)

cd AI/ReinforcementLearning
python rl_inference.py

3. Запустить GA-сервер (в другом терминале)

cd AI/GeneticAlgorithm
python ga_inference.py

4. Открыть проект в Unity 6

File → Open Project → выбрать папку BitBattles
Дождаться импорта ассетов

5. Запустить сцену

Assets/Scenes/MainMenu.unity
Нажать Play

Примечание: Если сервера не запущены, игра работает с ИИ по умолчанию.

## Результаты обучения

### RL-агент (управление юнитами)

| Метрика | Значение |
|---------|----------|
| Градиентных шагов | 55 325 |
| Игровых эпизодов | 1 070 |
| Win Rate | 51.6% |
| Replay Buffer | 50 000 |
| ε (начальный → финальный) | 0.5 → 0.01 |
| Максимальная награда за эпизод | +980.6 |

Анализ графиков:
- Loss стабилизировалась после 4000 шагов
- Q-values достигли пика на ~3000 шаге (модель взята в этой точке)
- Win Rate колеблется около 52%, что показывает равновесие с GA-противником

### GA-агент (расстановка врагов)

| Метрика | Значение |
|---------|----------|
| Поколений | 39 |
| Популяция | 15 |
| Max Fitness | 143.0 |
| Max Win Rate | 86.7% |
| Genetic Diversity | 0.08 → 0.32 |
| Mutation Power | 0.05 → 0.06 |

Анализ графиков:
- Fitness и Win Rate растут до 20-25 поколения, затем деградация из-за потери разнообразия
- Лучшая модель взята в диапазоне 20-25 поколений
